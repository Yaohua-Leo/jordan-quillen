# subplan007: Two-Step Cofibrant Replacement Toy For J3

## 0. Stage Goal

`subplan007` turns the extracted ChatGPT plan and the actual
`experiments/006-presentation-invariance-j3/results.json` output into the
next narrow experiment plan.

The target experiment directory is:

```text
experiments/007-genuine-cofibrant-replacement-j3/
```

The goal is to repair the Presentation B one-step diagnostic for:

```text
J_3 = (t)/(t^3)
```

by adding one explicit degree `2` generator that kills the extra naive
`H1` class seen in `subplan006`.

This stage is still a toy, hand-checkable low-degree replacement. It is not a
general cofibrant replacement algorithm for Jordan algebras.

## 1. Source Evidence

Experiment 006 has already run:

```text
python experiments/006-presentation-invariance-j3/run.py
```

Its current result records:

| Presentation | Computed H0 | Computed H1 | Status |
| --- | ---: | ---: | --- |
| `F(x)/(x^3)` | 1 | 1 | one-generator toy run |
| `F(x,y)/(y - x^2, xy, y^2)` | 1 | 2 | naive one-step diagnostic run |

The result file also records:

```json

  "presentation_invariance_verified": false,
  "presentation_B": {
    "cofibrant_replacement_status": "not implemented"
  }

```

The interpretation is that the extra `H1` class in Presentation B is not yet a
mathematical invariant. It is evidence that the naive one-step relation complex
has not included the higher syzygy data needed for a cofibrant replacement.

## 2. Scope Boundary

This stage may introduce only the minimal machinery needed to represent and
compute the following fixed indecomposable chain complex:

```text
Q2 -> Q1 -> Q0.
```

It may use pure Python exact rational linear algebra, preferably reusing the
existing local helpers from `src/jordan_qh/linear_algebra.py` and
`src/jordan_qh/quillen_toy.py` when that is natural.

This stage must not:

- implement automatic syzygy search;
- implement a general semi-free or cofibrant replacement engine;
- add Sage, SymPy, NumPy, or another dependency;
- overwrite or weaken Experiment 006, which remains the diagnostic failure
  case;
- claim full presentation invariance beyond the explicitly checked low
  degree range.

## 3. Mathematical Setup

Use Presentation B:

```text
F(x, y)/(r1, r2, r3)
```

where:

```text
r1 = y - x^2
r2 = x*y
r3 = y^2
```

Start with degree `0` generators:

```text
x, y
```

Attach degree `1` generators:

```text
a1, a2, a3
```

with differential:

```text
d(a1) = y - x^2
d(a2) = x*y
d(a3) = y^2
```

After applying indecomposables `Q(-)=(-)/(-)^2`, product terms vanish, so:

```text
d_Q(a1) = y
d_Q(a2) = 0
d_Q(a3) = 0
```

This explains the naive one-step Presentation B dimensions:

```text
H0 = 1
H1 = 2
```

The two naive `H1` generators are represented by `a2` and `a3`. The class
`a2` corresponds to the real truncated relation `x^3 = 0`; the class `a3`
comes from the redundant relation `y^2 = 0` and should be killed by a higher
cell.

## 4. Two-Step Correction

Attach one degree `2` generator:

```text
b
```

with:

```text
d(b) = a3 - x*a2 - (y + x^2)*a1 + x*(x*a1)
```

This is the fixed syzygy candidate extracted from the source HTML. For this
specific one-generated Jordan situation, its differential reduces to:

```text
d(d(b)) = x^2*x^2 - x*(x*x^2)
```

which vanishes by power-associativity in the one-generated Jordan subalgebra
generated by `x`.

This is an example-specific symbolic check, not an automatic Jordan syzygy
verifier.

After applying `Q`, all product terms vanish and:

```text
d_Q(b) = a3
```

The corrected indecomposable complex uses bases:

```text
Q0 = [x, y]
Q1 = [a1, a2, a3]
Q2 = [b]
```

with:

```text
d1(a1) = y
d1(a2) = 0
d1(a3) = 0
d2(b) = a3
```

Therefore the expected homology dimensions are:

| Degree | Expected dimension | Representative |
| --- | ---: | --- |
| `H0` | 1 | `x` |
| `H1` | 1 | `a2` |
| `H2` | 0 | none |

This agrees with Presentation A in degrees `0` and `1`, while explicitly
recording that full presentation invariance has not been proved.

## 7. Acceptance Criteria

This stage is complete only when:

1. The experiment directory records the two-step replacement setup and the
   warning that it is not a general algorithm.
2. The implemented chain complex satisfies `d_Q^2 = 0`.
3. The added generator `b` kills the extra naive `a3` class.
4. The computed dimensions are exactly `H0 = 1`, `H1 = 1`, and `H2 = 0`.
5. `results.json` is generated by `run.py`, not handwritten as a fake run.
6. The output explicitly compares the naive one-step diagnostic with the
   two-step correction.
7. Verification passes:

```text
python -m pytest
python -m ruff check .
python experiments/007-genuine-cofibrant-replacement-j3/run.py
python scripts/check_claims.py
```

## 8. Non-Goals

- Do not search for syzygies in arbitrary presentations.
- Do not extend beyond the fixed `J_3` Presentation B example.
- Do not mark a claim file as proved from this experiment alone.
- Do not identify intrinsic Jordan Quillen homology with TKK or Lie homology.
