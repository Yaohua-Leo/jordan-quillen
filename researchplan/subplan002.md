# subplan1: Run the first research workflow on a two-dimensional nilpotent Jordan algebra

## 0. Stage Goal

Stage 1 should not try to build the full Quillen homology theory of Jordan
algebras. Its purpose is to use one small example that is hand-checkable and
program-checkable to run the repository workflow once:

```text
mathematical definition draft -> hand expectation -> code implementation -> experiment record -> tests and claim check -> human review -> workflow feedback
```

The core deliverable is a worked example that has genuinely been run, while
making only modest claims. The result may support later definitions, but it
does not replace formal proofs about Beck modules, universal differentials, or
the Jordan cotangent complex.

## 1. Chosen Example

Work over a field `k` of characteristic `0`. Use the two-dimensional nonunital
nilpotent Jordan algebra

```text
J = k x + k y,
x * x = y,
x * y = 0,
y * y = 0.
```

Reasons for choosing this as the first example:

1. It tests multiplication, `J^2`, the derivation rule, and identity checks
   more meaningfully than a zero-multiplication algebra.
2. It avoids much of the noise from unital versus nonunital conventions.
3. It is the nonunital nilpotent part of the commutative associative algebra
   `k[x]/(x^3)`, so the Jordan identity has a transparent source.
4. It is small enough for hand calculation and pure-Python exact arithmetic.

## 2. Hand Expectations

The following calculations should be written in `theory/11-examples.md` or in
a dedicated worked-example note, and then checked against the experiment
output.

### 2.1 Jordan Identity

The multiplication is the restriction of a commutative associative
multiplication, so it satisfies the Jordan identity.

The repository currently has basis-level identity checks. Those checks should
be treated as smoke tests, not as proofs for arbitrary elements of an arbitrary
finite-dimensional algebra. For Stage 1, the minimum acceptable fix is to name
or document the basis-level check honestly. A multilinearized Jordan identity
check in characteristic `0` is useful, but it should not become a blocker for
running the first worked example.

### 2.2 Indecomposables

```text
J^2 = span_k{x*x, x*y, y*y} = span_k{y}.
```

Therefore

```text
Q(J) = J / J^2 ~= k . [x].
```

This is the most direct degree-zero shadow of absolute derived
abelianization. Do not claim that higher `H_n LQ(J)` has been computed unless
a cofibrant or simplicial resolution has actually been chosen and implemented.

### 2.3 Regular Derivations

Let `D: J -> J` be a linear map, and write

```text
D(x) = a x + b y,
D(y) = c x + d y.
```

The Leibniz rule and `y = x^2` give

```text
D(y) = D(x^2) = 2 x D(x) = 2 a y.
```

Hence `c = 0` and `d = 2a`. The relations `xy = 0` and `y^2 = 0` give no new
conditions. Thus

```text
Der_k(J,J) = { D_{a,b} | D(x) = a x + b y, D(y) = 2a y }.
```

With ordered basis `(x, y)` and column-vector convention, the matrix is

```text
[ a   0 ]
[ b  2a ]
```

The derivation space should have dimension `2`.

### 2.4 Provisional Differential Sanity Check

If `J` is temporarily viewed as the nonunital commutative associative/Jordan
example generated by `x` with relation `x^3 = 0`, the first-order differential
sanity check is

```text
dy = d(x^2) = 2 x dx,
0 = d(x^3) = 3 x^2 dx = 3 y dx.
```

In characteristic `0`, the provisional relation is `y dx = 0`, suggesting

```text
Omega_J ~= J . dx / (y . dx),
dy = 2 x dx.
```

This should be recorded only as a sanity check for the later construction of
`Omega_{J/A}`. It is not a Stage 1 acceptance blocker and must not be marked
as proved before the Beck-module convention is fixed.

## 5. Stage 1 Acceptance Criteria

Stage 1 is complete when:

1. The repository contains the concrete example
   `J = kx + ky`, `x^2 = y`, `xy = y^2 = 0`.
2. A command reproduces the multiplication table, `J^2`, `dim Q(J) = 1`, and
   `dim Der_k(J,J) = 2`.
3. `theory/11-examples.md` or an equivalent note contains the hand
   calculation.
4. `experiments/002-two-dimensional-jordan/results.md` separates computed
   results, provisional sanity checks, and uncomputed items.
5. `python -m pytest` and `python scripts/check_claims.py` pass; `ruff` also
   passes when available.
6. The final Codex summary distinguishes smoke tests, mathematical proofs, and
   statements that still depend on future definitions.

## 6. Explicit Non-Goals

Stage 1 should not:

- Build a full cofibrant or simplicial resolution.
- Claim higher Jordan Quillen homology has been computed.
- Compare with TKK Lie homology.
- Introduce Sage unless the pure-Python path fails and the reason is recorded
  in `PLAN.md`.
- Rewrite unread or partially read literature as settled conclusions.
- Treat a notebook as a durable result; results should land in
  `experiments/*/results.*` or `theory/`.
