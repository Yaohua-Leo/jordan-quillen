# subplan005: Quillen Derived Indecomposables Toy Benchmark

## 0. Stage Goal

`subplan005` turns the extracted ChatGPT plan into one narrow benchmark for
derived indecomposables in the sense of a toy Quillen-style abelianization
check.

The target experiment directory is:

```text
experiments/005-quillen-derived-indecomposables/
```

The goal is to record and eventually run the one-generator benchmark:

```text
F(x), F(x)/(x^2 - x), F(x)/(x^2), F(x)/(x^3), F(x)/(x^4).
```

This stage is not a full cofibrant replacement engine and does not prove the
Jordan Quillen homology theory.

## 1. Scope Boundary

This stage may introduce the minimal code needed for:

- ordinary indecomposables `Q(J) = J/J^2` for finite-dimensional algebras
  represented by structure constants;
- a one-generator toy relation model `F(x)/(r(x))`;
- a reproducible expected-value experiment for the five examples above.

This stage must not introduce Sage, SymPy, NumPy, or another dependency. Use
pure Python and exact rational arithmetic.

Do not fold the presentation-invariance issue for `J_3 = (t)/(t^3)` into this
stage. That belongs to `subplan006`.

## 2. Mathematical Convention

The toy model keeps only the linear part after applying `Q`.

If a semi-free one-relation presentation has:

```text
d(a) = r(x),
```

then the induced two-term complex after applying `Q` is:

```text
k a -> k x,
```

where the differential is multiplication by the coefficient of `x` in `r(x)`.

Consequences:

- If the linear coefficient is zero, then `dim H0 = 1` and `dim H1 = 1`.
- If the linear coefficient is nonzero, then `dim H0 = 0` and `dim H1 = 0`.
- The free case has no relation generator, so `dim H0 = 1` and `dim H1 = 0`.

Expected benchmark values:

| Example | Expected H0 | Expected H1 |
| --- | ---: | ---: |
| `F(x)` | 1 | 0 |
| `F(x)/(x^2 - x)` | 0 | 0 |
| `F(x)/(x^2)` | 1 | 1 |
| `F(x)/(x^3)` | 1 | 1 |
| `F(x)/(x^4)` | 1 | 1 |

## 4. Experiment Record

`expected.json` should contain the five expected examples and the dimensions
above.

`results.json` must not fabricate computation. Before the toy module is
implemented and run, it should say:

```json
{
  "status": "not run yet"
}
```

After implementation, `run.py` may write a result with:

```json
{
  "status": "run",
  "passed": true,
  "examples": []
}
```

where `examples` contains the computed dimensions and comparison against
`expected.json`.

## 5. Acceptance Criteria

This stage is complete only when:

1. The source implementation computes `Q(J)=J/J^2` by exact linear algebra.
2. The one-generator toy model records the linear coefficient convention.
3. The five benchmark examples produce the expected `H0` and `H1` dimensions.
4. `results.json` is generated by `run.py`, not handwritten as a fake run.
5. Verification passes:

```text
python -m pytest
python -m ruff check .
python experiments/005-quillen-derived-indecomposables/run.py
python scripts/check_claims.py
```

## 6. Non-Goals

- Do not implement full free Jordan algebras.
- Do not implement general cofibrant replacements.
- Do not claim presentation invariance.
- Do not identify intrinsic Jordan Quillen homology with TKK or Lie homology.
- Do not promote the toy computation to a theorem.
