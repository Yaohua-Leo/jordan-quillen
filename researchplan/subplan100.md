# subplan100: Experiment 100 For Computing One-Generator Cofibrant Cells

## 0. Plan-Only Boundary

This file is only an experiment plan. It does not run Experiment 100, create a
new experiment directory, write result files, modify earlier experiment
outputs, or change any claim status.

The future experiment target is:

```text
experiments/100-square-zero-nonunital-cofibrant-weight10/
```

Experiment 100 is a bounded computation of the low-degree cell layers for the
one-generator nonunital square-zero Jordan algebra. The plan must not know the
answer for `tilde J_2` in advance. In particular, it must not prefill a
degree-2 generator, a differential formula, raw cycle counts, or a
minimal-cell table from the browser conversation.

The experiment is not a general cofibrant replacement engine, does not apply
indecomposables `Q`, and must not claim a theorem about the infinite
resolution.

## 1. Source Context And Trust Boundary

The browser-side conversation fixes only the experiment question:


Any predicted answer in that conversation is deliberately excluded from this
plan. The future runner must compute the cells and differentials from the

## 2. Mathematical Object

Work over:

```text
QQ
```

with the nonunital Jordan algebra:

```text
J = Jord<x> / (x^2).
```

Equivalently, `J` is the one-dimensional vector space spanned by the class of
`x`, with zero product:

```text
x^2 = 0.
```

Use the presentation:

```text
F_Jord(x) -> J
x |-> x_bar
```

The relation `x^2 = 0` is part of the input presentation, but the future
experiment should still record relation-killing degree-1 cells as computed
output, not as handwritten result data.

The computation is bounded by:

```text
max_weight = 10
max_homological_degree = 2
```

## 3. Fixed Conventions

Use product weight:

```text
wt(x) = 1
wt(ab) = wt(a) + wt(b)
```

For any cell discovered later, define:

```text
wt(cell) = wt(d(cell))
```

unless a future implementation records a stricter convention with a reason.

Use homological degrees:

```text
|x| = 0
|degree-1 relation cells| = 1
|degree-2 syzygy cells| = 2
```

The differential must preserve product weight:

```text
wt(d(a)) = wt(a).
```

Use the signed derivation rule:

```text
d(ab) = d(a)b + (-1)^|a| a d(b).
```

Use this matrix convention wherever possible:

```text
source_rows_target_coordinates
```

For a map `d: S -> T`, each row corresponds to a basis element of `S`, and the
row entries are coordinates in the chosen basis of `T`.

### 3.1 Backend Convention Audit

The default committed runner should use exact rational arithmetic. If it reuses
the existing low-weight Jordan backend, the output must state which convention
is being used for products involving odd-degree generators and for the signed
derivation rule.

At minimum, record:

```text
category = nonunital dg Jordan algebras
base_field = QQ
arithmetic = exact_rational
differential_rule = signed_derivation
graded_sign_convention = documented
```

If the implemented backend treats products as ordinary commutative
nonassociative Jordan products with a signed derivation differential, state
that explicitly. A separate theory check is required before interpreting the
bounded backend as a complete graded operadic dg Jordan computation.

## 4. Stage Goal

For weights:

```text
1 <= w <= 10
```

the future experiment should compute the bounded cell layers:

```text
V0,w
V1,w
V2,w
```

where:

- `V0` contains the degree-0 generators needed to map onto `J`;
- `V1` contains relation cells needed to kill the kernel of the degree-0
  presentation map, modulo the closure generated by earlier accepted relation
  cells;
- `V2` contains syzygy cells represented by nonzero classes in
  `H1(A^(1))`, where `A^(1)` is the dg Jordan algebra after adjoining the
  computed `V1` cells.

The primary result is not a predicted formula for `tilde J_2`; it is the
computed list of degree-2 cells and their certified differentials for completed
weights.

Allowed result language:

```text
Experiment 100 computed the listed V0, V1, and V2 cells for completed weights
w <= 10 under the implemented bounded dg Jordan convention.
```

Forbidden result language:

```text
Experiment 100 confirms a predicted tilde J_2.
Experiment 100 proves the full cofibrant replacement of J.
No further cells occur in all weights.
The infinite resolution terminates after the listed cells.
```

## 5. Algorithm

The computation is weight-blocked. Each weight should be processed with exact
rational arithmetic and deterministic quotient bases.

### 5.1 Enumerate Degree-0 Terms And The Presentation Map

Enumerate homogeneous terms in the free nonunital Jordan algebra:

```text
A^(0) = F_Jord(x)
```

through weight `10`, modulo the linearized Jordan identity used by the local
backend.

Construct the presentation map:

```text
epsilon: A^(0) -> J
```

by:

```text
epsilon(x) = x_bar
epsilon(product of length >= 2) = 0
```

For each weight `w`, compute:

```text
K0,w = ker(epsilon_w).
```

### 5.2 Compute Degree-1 Relation Cells `V1`

For each weight `w`, compute the part of `K0,w` not already generated by the
Jordan ideal closure of the differentials of accepted lower or same-stage
relation cells.

Attach one degree-1 cell for each selected basis element:

```text
r1_{global_index:05d}_w{w}
```

with:

```text
|r1_i| = 1
wt(r1_i) = w
d(r1_i) = selected relation in K0,w
```

The runner must record:

- the selected relation formula;
- coordinates in the degree-0 quotient basis;
- a certificate that the relation lies in `K0,w`;
- a certificate that the relation is independent modulo the closure of earlier
  accepted relation differentials.

Although the input presentation contains `x^2 = 0`, the result object must
still treat the corresponding relation cell, if selected, as computed output.

### 5.3 Build The First-Layer dg Jordan Algebra

After computing `V1` through the requested weights, form:

```text
A^(1) = F_dgJord(V0, V1)
```

with differential determined by the computed `V1` records and the signed
derivation rule.

For each weight `w`, construct:

```text
C0,w = A^(1)_{0,w}
C1,w = A^(1)_{1,w}
C2_old,w = A^(1)_{2,w}
```

Here `C2_old,w` means degree-2 elements built from old generators `V0` and
`V1` only. It does not include any new `V2` cells.

Construct:

```text
d1,w : C1,w -> C0,w
d2_old,w : C2_old,w -> C1,w
```

With the source-row matrix convention, the chain condition should be checked
as:

```text
d2_old_matrix @ d1_matrix = 0
```

or the equivalent local helper call.

### 5.4 Compute Degree-1 Homology Before Attaching `V2`

For each weight `w`, compute:

```text
Z1,w = ker(d1,w)
B_old,w = im(d2_old,w)
H1_old,w = Z1,w / B_old,w
```

The runner-friendly name `H1_old,w` is not a new homology theory. It only means
first homology in `A^(1)` before adjoining any new degree-2 cells.

Record:

```text
dim C0,w
dim C1,w
dim C2_old,w
rank d1,w
rank d2_old,w
dim Z1,w
dim B_old,w
dim H1_old,w
```

Verify:

```text
dim Z1,w = dim C1,w - rank d1,w
dim B_old,w = rank d2_old,w
dim H1_old,w = dim Z1,w - dim B_old,w
```

### 5.5 Compute Degree-2 Cell Candidates `V2`

For every nonzero class:

```text
[z_i] in H1_old,w
```

the future experiment should attach a formal degree-2 cell:

```text
s2_{global_index:05d}_w{w}
```

with:

```text
|s2_i| = 2
wt(s2_i) = w
d(s2_i) = z_i
```

This is the computed `V2` layer through the requested weights. The plan does
not prescribe how many such cells appear, what weights they appear in, or what
their formulas are.

Every accepted representative `z_i` must include:

- a full formula in the chosen quotient basis;
- coordinates in the `C1,w` basis;
- a check that `d1,w(z_i) = 0`;
- a mandatory non-boundary certificate proving `z_i not in B_old,w`;
- a quotient-independence certificate modulo `B_old,w` and previously selected
  representatives.

### 5.6 Stop After Degree 2

Do not attach the computed `s2_i` and continue to `V3`. Do not compute:

```text
H2_new
V3
V4
...
```

Do not apply:

```text
Q(A^(1))
Q(tilde J)
```

The primary output is the bounded list of computed `V0`, `V1`, and `V2` cells
through weight `10`.

## 6. Mandatory Certificates

### 6.1 Degree-1 Relation Certificates

Every reported relation cell `r1_i` must prove:

```text
d(r1_i) in ker(epsilon)
```

and must record why its differential is independent from the closure generated
by previously accepted relation-cell differentials.

If the implementation cannot certify minimality of the selected `V1` basis, it
must mark the result as:

```text
minimality_certified = false
```

and use cautious interpretation language.

### 6.2 Cycle Certificates For `V2`

Every reported degree-2 cell `s2_i` must prove:

```text
d1,w(d(s2_i)) = 0.
```

The normal form of `d1,w(d(s2_i))` in `C0,w` must be recorded as zero.

### 6.3 Non-Boundary Certificates For `V2`

Every reported representative `z_i = d(s2_i)` must prove:

```text
z_i not in B_old,w.
```

The mandatory certificate is an exact RREF remainder:

```text
rem_B_old,w(z_i) != 0.
```

The record should contain:

```json
{
  "certificate_type": "rref_remainder",
  "B_old_rref_pivots": ["..."],
  "boundary_remainder_mod_B_old": "...",
  "boundary_remainder_nonzero": true,
  "not_in_B_old": true
}
```

If the RREF basis is large, store the full sparse RREF in a cache file and
record a stable checksum or cache path in the cell record.

### 6.4 Quotient Independence Certificates

Within each weight, the selected `V2` representatives must be independent in:

```text
Z1,w / B_old,w.
```

Equivalently:

```text
dim(B_old,w + <z_1, ..., z_n>) = dim B_old,w + n.
```

Also verify:

```text
n = dim H1_old,w.
```

## 9. Acceptance Criteria For Future Experiment 100

The future experiment has a useful completed run only when:

1. The experiment directory points back to `researchplan/subplan100.md`.
2. The runner uses exact rational arithmetic over `QQ`.
3. The runner records the dg Jordan backend and sign convention.
4. `expected.json` contains setup only and does not prefill computed cells or
   predicted results.
5. Every reported `V1` cell has a certificate that its differential lies in
   the presentation kernel.
6. Every completed weight records `dim C0,w`, `dim C1,w`, `dim C2_old,w`,
   `rank d1,w`, `rank d2_old,w`, `dim Z1,w`, `dim B_old,w`, and
   `dim H1_old,w`.
7. Every reported `V2` differential is certified to be a cycle.
8. Every reported `V2` differential has a mandatory RREF remainder certificate
   showing that it is not in `B_old,w`.
9. Reported `V2` representatives are independent modulo `B_old,w`.
10. The number of reported `V2` cells in each completed weight equals
    `dim H1_old,w`.
11. `results.json` distinguishes completed, failed, skipped, and not-tested
    weights.
12. The report labels the output as bounded computational evidence only.
13. The report contains no theorem-like statement about the infinite
    cofibrant replacement.
