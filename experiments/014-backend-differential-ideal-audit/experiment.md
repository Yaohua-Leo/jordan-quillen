# Experiment 014: Backend Differential And Multiplicative-Ideal Audit

Plan: `researchplan/subplan014.md`

Status: implemented runner

## Question

EXP014 audits whether the current `low_weight_jordan` ordinary quotient
backend is stable enough to be treated as a dg Jordan quotient in a bounded
range.  It checks the backend relation rows themselves, their signed
derivation differentials, and their products with bounded quotient-basis
multipliers.

## Method

The experiment uses the same first-layer model as EXP011:

```text
A^(1)=Jord<x,y,r_xx,r_xy,r_yy>
d(r_xx)=x^2, d(r_xy)=xy, d(r_yy)=y^2, d(x)=d(y)=0
```

For every homogeneous relation row `R` generated by `low_weight_jordan`, the
runner checks:

```text
reduce(R) = 0
reduce(d(R)) = 0
reduce(R*m) = 0
```

The multiplier `m` ranges over quotient basis terms whose total degree and
weight stay inside the requested bounds.

## Interpretation Boundary

This is a bounded backend audit only.  It does not apply `Q`, does not attach
or construct `V3+` cells, and does not assert a global theorem.  A failed audit
means EXP011/EXP013 should be interpreted as backend-relative diagnostics until
the quotient convention is repaired or separately justified.
